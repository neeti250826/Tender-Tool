NUPCO Tender Automation Workflow

Operational flow chart showing the full workflow from tender discovery to final consolidated awarded-tender output, with a practical view of what AI can automate fully, what needs partial handling, and where manual review is still useful.

High Automation Partial Automation Manual Review / Risk Point

Overall View

Best Automation Fit

Discovery, status filtering, tender URL capture, PDF download, structured parsing, normalization, and output generation can all be automated well.

Main Risk Areas

PDF layout changes, wrapped rows, missing item-list attachments, and occasional manual validation of award interpretation remain the main risk points.

Captcha / OCR Need

For the current NUPCO awarded workflow, no captcha bypass is needed. OCR is not the default requirement; it is only needed if a critical attachment becomes image-only.

Flow Chart

-Stage 1: Site Discovery High Automation

Open the NUPCO tenders page and identify how the listing behaves.

What To Learn

  • Main listing URL
  • Whether the page uses pagination or infinite scroll
  • Visible status filters
  • How tender cards expose status and tender ID

AI Can Automate

  • Open the page and inspect the DOM
  • Detect scroll-based loading
  • List visible status filters
  • Capture tender-card links and statuses

Manual Check

  • Confirm the business meaning of each status label
  • Confirm whether the workflow should include both First Result and Final Result
Stage 2: Filtering For Awarded Tenders High Automation

Reduce the tender universe to award-relevant statuses only.

Key Filters

  • Final Result
  • First Result

AI Can Automate

  • Apply status filters in the browser
  • Scroll until no new cards load
  • Build a queue of awarded-tender candidate URLs

Manual Check

  • Decide whether to exclude preliminary-only results
  • Confirm if direct purchase should be included in a separate process
Stage 3: Tender Detail Discovery High Automation

Visit each tender detail page and discover where the actual awarded data lives.

What To Inspect

  • Tender ID
  • Tender title
  • Status
  • Publication or bid-opening date
  • Submission deadline
  • Attachment links

AI Can Automate

  • Open each detail page
  • Extract core metadata fields
  • Detect PDF attachment buckets such as item list, preliminary result, and final result

Manual Check

  • Confirm unusual attachment labels
  • Confirm whether a PDF is actually a result file or just a general notice
Stage 4: Item-Level Detail Access Partial Automation

Determine whether awarded item rows, quantities, and UOM are available, and from which attachment.

Possible Sources

  • Item List PDF
  • Result PDF
  • Wrapped tabular text inside attachments

AI Can Automate

  • Download likely item-list attachments
  • Parse table-like text from PDFs
  • Rebuild wrapped rows into structured item lines

Risk Points

  • PDFs can have inconsistent layouts
  • Item lists may be separate from award-result files
  • Some tenders may not expose full item details at all
Stage 5: Download Attachments High Automation

Fetch all result-related and item-related PDFs for each tender.

Files To Download

  • Final Result PDFs
  • Preliminary Result PDFs
  • Item List PDFs

AI Can Automate

  • Retry transient download failures
  • Classify files by name patterns
  • Keep a debug log of missing or failed files

Manual Check

  • Only needed when attachment naming is ambiguous
Stage 6: Parse Awarded Data Partial Automation

Extract supplier names, awarded dates, amounts, item descriptions, UOM, and quantities from the PDFs.

AI Can Automate

  • Text extraction from normal PDFs
  • Row reconstruction from table-like text
  • Merge item rows with supplier result rows by item number

Where OCR Might Be Needed

  • Only if a critical PDF is image-based or scanned
  • Not required for the current normal NUPCO flow in most tested cases

Manual Review Points

  • Supplier mapping when PDFs are messy
  • Award-date interpretation if multiple dates appear
  • Tenders with missing item rows
Stage 7: Normalize And Consolidate High Automation

Transform extracted content into one clean row format across all tenders.

Normalization Work

  • Standardize dates
  • Standardize currency to SAR
  • Populate tender-level and item-level fields
  • Build dedup keys

AI Can Automate

  • Schema mapping
  • Null handling
  • Fallback logic
  • Per-tender row generation

Manual Check

  • Spot-check output quality only
Stage 8: Validation And Delivery High Automation

Validate the output files and prepare the final delivery set.

AI Can Automate

  • JSONL output
  • CSV output
  • Row validation
  • Debug and error logs

Useful Checks

  • Total row count
  • Missing supplier names
  • Missing item quantities
  • Unexpected empty tenders

Manual Review

  • Final business-level QA on a sample of tenders

Automation Assessment Table

Workflow Area Can AI Automate? Comments
Find listing URL and filters Yes Stable and straightforward.
Handle pagination / scroll loading Yes NUPCO uses scroll-based loading; this is automatable.
Filter for awarded tender statuses Yes Final Result and First Result are the main award-relevant filters.
Open tender detail pages Yes Direct browser automation works.
Detect attachment types Yes Works well with naming rules plus fallback heuristics.
Download result and item PDFs Yes Retry logic is important for intermittent failures.
Extract item-level details Partially Depends on attachment quality and whether item PDFs exist.
Consolidate supplier rows with item rows Partially Usually automatable, but some tenders may need logic tuning.
OCR for scanned documents Sometimes Only required if a key PDF is image-only.
Captcha hacking No Not part of the current NUPCO awarded workflow.
Practical conclusion: NUPCO awarded-tender extraction is highly automatable end-to-end. The only areas that may still need occasional manual review are attachment interpretation, malformed PDF layouts, and rare cases where item-level details are missing or only partially available.

Recommended Delivery Model

Fully Automated Layer

  • Listing discovery
  • Status filtering
  • Tender queue creation
  • Attachment download
  • Schema output to JSONL and CSV

Assisted AI Layer

  • PDF parsing with retry and heuristics
  • Item-supplier row merging
  • Fallback logic for sparse tenders

Manual QA Layer

  • Sample-based review of final awarded outputs
  • Inspection of tenders with very low parsed row counts
  • Inspection of tenders where item details are unexpectedly empty
